# Как обновить дашборд через GitHub Desktop

## Вариант А. Просто заменить старые файлы новой версией

1. Распакуй архив `bitrix-task-dashboard-static-v2.zip`.
2. Открой GitHub Desktop.
3. Выбери репозиторий дашборда.
4. Нажми `Repository` → `Show in Explorer` / `Show in Finder`.
5. Скопируй все файлы из новой распакованной папки в корень репозитория поверх старых файлов.
6. Согласись на замену файлов.
7. В GitHub Desktop в поле Summary напиши:

```text
Update static dashboard rules and upload
```

8. Нажми `Commit to main`.
9. Нажми `Push origin`.
10. Открой GitHub Pages-ссылку дашборда и обнови страницу.

В корне репозитория должны лежать:

```text
index.html
README.md
GITHUB_DESKTOP_INSTRUCTION.md
assets/
data/
.nojekyll
```

## Что удалить, если осталось от старой версии

Если в репозитории остались файлы от версии на Streamlit/Python, они больше не нужны:

```text
app.py
requirements.txt
process_exports.py
src/
config/
reports/
.github/
```

Их можно удалить, затем сделать commit и push.

## Как включить GitHub Pages

1. Открой репозиторий на GitHub в браузере.
2. Перейди в `Settings`.
3. В левом меню открой `Pages`.
4. В блоке `Build and deployment` выбери:
   - Source: `Deploy from a branch`
   - Branch: `main`
   - Folder: `/root`
5. Нажми `Save`.
6. Подожди 1–3 минуты.
7. Открой ссылку, которую GitHub покажет в разделе Pages.

## Как быстро проверить новую выгрузку без коммита

1. Открой опубликованный дашборд.
2. Нажми `Загрузить выгрузку`.
3. Выбери новый `.xls` из Битрикс24.
4. Дашборд сразу пересчитает задачи в браузере.

Важно: такая загрузка сохраняется только в твоем браузере. Для постоянного хранения и доступа другим людям файл нужно добавить в репозиторий.

## Как добавить новую выгрузку в репозиторий

1. Положи файл в:

```text
data/raw/
```

2. Имя файла должно быть таким:

```text
tasks_YYYY-MM-DD_HH-MM.xls
```

3. Открой:

```text
data/exports.json
```

4. Добавь имя новой выгрузки в список.

Пример:

```json
{
  "files": [
    "tasks_2026-07-03_07-04-17.xls",
    "tasks_2026-07-03_11-09-38.xls",
    "tasks_2026-07-04_09-00.xls"
  ]
}
```

5. В GitHub Desktop:
   - Summary: `Add Bitrix export 2026-07-04 09-00`
   - `Commit to main`
   - `Push origin`

## Как часто прикладывать выгрузки

Обычный режим:

```text
1 раз в рабочий день утром до 09:30
```

Активная проектная фаза:

```text
2 раза в день: утром до 09:30 и после обеда в 14:00–15:00
```

Критический период перед сдачей:

```text
3 раза в день: утром, в середине дня и в конце рабочего дня
```

Старые выгрузки не удаляй. Они нужны для экрана `Динамика`.
