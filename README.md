# ITO Mission Control

Статический action-only дашборд для контроля задач инженерно-технического отдела из выгрузок Битрикс24.

Работает как обычный сайт на GitHub Pages:

- без Streamlit;
- без Python;
- без сервера;
- без сборки;
- без установки зависимостей.


## v9: no-noise / ultra high-tech

В этой версии дашборд переведен в режим максимальной конкретики:

- по умолчанию скрываются зеленые и справочные задачи;
- в основных таблицах остается только `Сигнал → Кому писать → Кому мяч → Что сделать`;
- раздел `Контроль сегодня` стал коммуникационным маршрутизатором;
- раздел `Люди` показывает только сотрудников, где есть действие или риск;
- раздел `Проекты` показывает только проекты с управленческим сигналом;
- раздел `Динамика` показывает только сигнальные изменения: просрочки, контроль, сроки, ответственных, рост риска;
- раздел `Action-only задачи` заменяет общий шумный список задач;
- дизайн переработан в темный high-tech стиль с glass-панелями, neon-акцентами, плотными карточками и фокусом на приоритетах.


## v10: Операционный Гант

Добавлен отдельный рабочий раздел `Операционный Гант`, который строится прямо в браузере по полям выгрузки Битрикс24:

- плановая полоса — когда заполнены `Планируемая дата начала` и `Планируемая дата окончания` / крайний срок;
- рабочая полоса — когда есть `Дата начала работы` и крайний срок;
- ромб-событие — когда у задачи есть только дедлайн;
- отдельный сигнал `НЕТ ДАТ` — когда задача не имеет стартовой даты и срока;
- красный пунктирный хвост — период просрочки от дедлайна до даты текущего среза.

Доступны горизонты 30, 90, 180 дней и календарный год, группировка по проектам, ответственным или статусам, сортировки по риску / сроку / старту, сворачивание групп, компактный режим, быстрый возврат к сегодняшней дате и CSV-выгрузка текущего представления. Карточки над шкалой работают как быстрые фильтры и показывают качество планирования: сколько задач имеют интервал, сколько представлены только дедлайном, сколько не имеют срока и сколько уже просрочены.

## Что изменено в этой версии

- Дашборд растянут на всю ширину монитора.
- Задачи со статусом `Отложена` полностью исключаются из расчетов, таблиц, рисков, загрузки людей и динамики.
- Проектные контейнеры исключаются из расчетов как самостоятельные задачи. Их подзадачи учитываются внутри соответствующих проектов.
- Ежедневные задачи Максима исключаются из расчетов:
  - `Ежедневная проверка статусов закупок/договоров`;
  - `Ежедневный контроль выполнения задач`.
- Добавлена загрузка новой выгрузки прямо через сайт кнопкой `Загрузить выгрузку`.
- После каждой загрузки строится отдельный автоотчет: KPI-дельты, управленческое резюме, журнал движения задач, топ рисков по людям и проектам.
- Обновлен раздел `Дайджесты`: теперь это рабочий отчет без лишнего шума — роль, причина, кому сейчас мяч, следующее действие, лимиты по блокам и визуальный мини-отчет.
- Логика дайджестов стала ролевой: исполнителю показывается только то, что он может сделать; постановщику — приемка; наблюдателям — контроль только после 3 дней или при рисках; Дмитрию Храпугину — управленческие эскалации отдела.
- У каждого сотрудника есть готовые форматы для пересылки: визуальный мини-отчет, `Печать / PDF`, HTML-файл, краткий текст для мессенджера, подробный текст для почты / Битрикс24, TXT-файл и копирование отдельной карточки задачи.
- Кнопка `Скачать CSV` в разделе дайджестов выгружает все персональные пункты по сотрудникам, ролям и задачам.
- Добавлено сквозное правило контроля: наблюдатели подключаются к задачам `Ждёт контроля` после 3 дней, красная зона приемки включается после 5 дней, исполнитель не краснеет за зависшую приемку.
- Кнопка `Скачать отчет CSV` выгружает сводку, инсайты и журнал изменений между двумя последними срезами.
- При загрузке файла через сайт дашборд сразу открывает именно загруженную выгрузку, даже если имя файла не содержит дату.
- В проект добавлена новая выгрузка `tasks_2026-07-08_10-43-28.xls`.

## Правило по задачам в статусе “Ждёт контроля”

Если задача находится в статусе `Ждёт контроля`, дашборд считает, что исполнитель уже передал результат на проверку. Поэтому задача не должна попадать исполнителю в красную зону за зависшую приемку.

Пороги:

- `0–2 дня` — мяч у постановщика: нужно проверить, принять или вернуть;
- `3–4 дня` — подключаются наблюдатели, задача появляется в блоке `Нужно проверить / принять`;
- `5+ дней` — красная зона приемки: постановщик / наблюдатели должны снять зависание;
- `Дмитрий Храпугин` получает не обычный список наблюдателя, а управленческие эскалации: долгий контроль, долгая просрочка, красный риск, отсутствие активности.

В таблицах `Кому писать и что сделать`, `Приемка и контроль` и `Action-only задачи` добавлены поля `Кому писать`, `Кому мяч` и `Что сделать`. Это правило применено на уровне общей модели задачи, поэтому влияет на дайджесты, загрузку людей, надежность, CSV-экспорты, список пуша и контрольный inbox.

## Как открыть

После загрузки репозитория на GitHub включи GitHub Pages:

`Settings` → `Pages` → `Deploy from a branch` → `main` → `/root` → `Save`.

После публикации открой ссылку GitHub Pages. Главная страница дашборда — `index.html`.

## Быстрая загрузка выгрузки через сайт

На опубликованном дашборде нажми сверху `Загрузить выгрузку` и выбери файл `.xls` из Битрикс24.

Такой файл будет обработан прямо в браузере. Это удобно для быстрой проверки без GitHub Desktop.

После загрузки сверху появится/обновится блок `Что требует действия`. Он сравнит текущий файл с предыдущей доступной выгрузкой и покажет:

- новые просрочки и снятые просрочки;
- зависшую приемку и задачи `Ждёт контроля`;
- изменения сроков, ответственных и рост риска;
- кому писать первым;
- конкретное следующее действие.

Важно: загрузка через сайт сохраняет файл только в браузере пользователя. Чтобы выгрузка была доступна всем и не пропадала после очистки браузера, добавь файл в репозиторий через GitHub Desktop.

## Как добавить выгрузку в репозиторий

Новые выгрузки Битрикс24 класть в папку:

```text
data/raw/
```

Формат имени файла:

```text
tasks_YYYY-MM-DD_HH-MM.xls
```

Примеры:

```text
tasks_2026-07-03_09-00.xls
tasks_2026-07-03_14-00.xls
tasks_2026-07-04_09-00.xls
```

После добавления файла нужно открыть:

```text
data/exports.json
```

И добавить имя новой выгрузки в список `files`.

Пример:

```json
{
  "files": [
    "tasks_2026-07-03_07-04-17.xls",
    "tasks_2026-07-03_11-09-38.xls",
    "tasks_2026-07-03_14-00.xls"
  ]
}
```

## Частота обновления

Обычный режим: 1 раз в рабочий день утром до 09:30.

Активная проектная фаза: 2 раза в день — утром и после обеда.

Критический период перед сдачей: 3 раза в день — утром, в середине дня и в конце рабочего дня.

Старые выгрузки не удалять: они нужны для раздела `Динамика` и для блока `Изменения с прошлой выгрузки` внутри персональных дайджестов.

## Персональные дайджесты

Раздел `Дайджесты` строится по всем ролям из выгрузки Битрикс24:

- `Исполнитель` / `Ответственный` — что человек должен сделать сам;
- `Постановщик` — что нужно принять, проконтролировать или вернуть;
- `Соисполнители` — где нужен вклад сотрудника;
- `Наблюдатели` — не все подряд: задачи на контроле только с 3-го дня, красная зона с 5-го дня, плюс долгие просрочки и красные риски;
- `Дмитрий Храпугин` — отдельная управленческая роль руководителя отдела: только эскалации, системные зависания и красные риски.

Для каждого сотрудника доступен фильтр:

- `Красная зона, проверка и сегодня` — оптимальный ежедневный режим без информационного шума;
- `Только красная зона` — короткий список для срочного пуша;
- `Все пункты` — полный список, включая ближайшие сроки, гигиену и изменения.

Интерфейс дайджестов теперь состоит из трех рабочих зон:

- `Кому писать первым` — список сотрудников с быстрыми счетчиками: красная зона, нужно проверить, сегодня, всего пунктов.
- `Рабочая карточка` — визуальные карточки задач с ролью сотрудника, причиной попадания, сроком, риском, полем `Кому сейчас мяч` и конкретным действием. В каждом блоке есть лимит задач, чтобы дайджест не превращался в полотно.
- `Мини-отчет для пересылки` — красивый визуальный отчет, который можно открыть в отдельном окне, отправить через `Печать / PDF` или скачать как HTML, плюс ниже остается готовый текст для чата.

Доступны способы пересылки:

- `Открыть мини-отчет` — отдельное красивое окно с персональным мини-отчетом;
- `Печать / PDF` — быстро сохранить мини-отчет в PDF для отправки коллегам;
- `Скачать HTML` — отдельный HTML-файл, который можно переслать как вложение;
- `Скопировать кратко` — компактный текст для Telegram, Битрикс24-чата или другого мессенджера;
- `Скопировать подробно` — расширенный текст для почты, комментария в Битрикс24 или управленческого разбора;
- `Скачать TXT` — файл с текущим дайджестом;
- `Скопировать задачу` на карточке — отдельный блок по конкретной задаче, если не нужно отправлять весь дайджест.
