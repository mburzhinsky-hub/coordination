# Контроль задач ИТО из выгрузок Битрикс24

Профессиональный Streamlit-дашборд для координатора инженерно-технического отдела.  
Цель — быстро видеть, кого нужно пушить, какие задачи просрочены, что ждёт контроля, где нарушена дисциплина ведения задач и на кого из сотрудников можно рассчитывать.

## Что показывает дашборд

- Контроль сегодня: KPI и пуш-лист.
- Загрузка людей: индекс загрузки, надежности и доступности.
- Проекты: риск по родительским задачам / проектам.
- Ждёт контроля: отдельный inbox задач, которые нужно принять, закрыть или вернуть.
- Гигиена задач: нет срока, нет родителя, нет активности, отложена и просрочена.
- Динамика: новые задачи, исчезнувшие задачи, изменения сроков, новые просрочки.
- Аналитика: распределение по статусам и рискам.

## Быстрый старт локально

```bash
python -m venv .venv
source .venv/bin/activate  # macOS/Linux
# .venv\Scripts\activate   # Windows
pip install -r requirements.txt
python process_exports.py
streamlit run app.py
```

После запуска откройте адрес, который покажет Streamlit, обычно:

```text
http://localhost:8501
```

## Куда класть выгрузки Битрикс24

Новые выгрузки кладутся в папку:

```text
data/raw/
```

Формат имени:

```text
tasks_YYYY-MM-DD_HH-MM.xls
```

Примеры:

```text
tasks_2026-07-03_09-00.xls
tasks_2026-07-03_14-00.xls
tasks_2026-07-04_09-00.xls
```

Старые выгрузки не удаляйте. Они нужны для экрана динамики.

## Частота обновления

Обычный режим:

```text
1 раз в рабочий день утром, до 09:30.
```

Активная проектная фаза:

```text
2 раза в день: утром до 09:30 и после обеда в 14:00–15:00.
```

Критический период перед сдачей проекта:

```text
3 раза в день: утром, в середине дня и в конце рабочего дня.
```

Минимальный регламент:

1. Открыть Битрикс24.
2. Перейти в список задач отдела.
3. Проверить, что в список попадают активные задачи, просроченные задачи, задачи на контроле и задачи без срока.
4. Экспортировать список задач в `.xls`.
5. Переименовать файл по шаблону `tasks_YYYY-MM-DD_HH-MM.xls`.
6. Положить файл в `data/raw/`.
7. Запустить `python process_exports.py` или закоммитить файл в GitHub, если настроен workflow.
8. Открыть дашборд и начать с разделов: `Контроль сегодня`, `Загрузка людей`, `Ждёт контроля`, `Гигиена задач`.

## Как работает риск задачи

Для каждой задачи рассчитывается `risk_score` от 0 до 100.

Основные факторы:

- просрочка;
- глубина просрочки;
- срок сегодня;
- срок в ближайшие 3 дня;
- статус `Ждёт контроля`;
- отсутствие крайнего срока;
- отсутствие активности больше 7/14 дней;
- отложенная, но просроченная задача;
- отсутствие родительской задачи;
- отсутствие проекта.

Пороговые значения и веса находятся в:

```text
config/dashboard_rules.yaml
```

## Как работает загрузка людей

Если в выгрузке нет оценок трудозатрат, используется прокси-модель:

```text
workload_score =
  active_tasks * 1
  + overdue_tasks * 2.5
  + due_today_tasks * 2
  + due_soon_tasks * 1.5
  + in_progress_tasks * 1.2
  + waiting_control_tasks * 0.8
  + no_deadline_tasks * 0.7
```

Категории:

```text
0–5       низкая загрузка
6–12      нормальная загрузка
13–20     высокая загрузка
21+       перегруз
```

## Как работает доступность сотрудников

Доступность — это ответ на вопрос: “На кого можно рассчитывать для новых задач?”

Она учитывает:

- текущую загрузку;
- надежность ведения задач;
- количество просрочек;
- срочные задачи;
- задачи без срока;
- зависшие задачи без активности.

Важно: человек не считается свободным только потому, что у него мало задач. Если эти задачи просрочены, без срока или без активности, индекс доступности снижается.

## Безопасность

Выгрузки Битрикс24 могут содержать ФИО, названия клиентов, договоры и внутреннюю информацию.

Рекомендуемый режим:

- реальное использование — только приватный GitHub-репозиторий;
- публичный GitHub — только код и обезличенные демо-данные;
- реальные файлы из `data/raw/` не публиковать в открытом доступе.

## GitHub Actions

Workflow находится здесь:

```text
.github/workflows/process_exports.yml
```

Он запускается при изменениях в `data/raw/` и вручную через `workflow_dispatch`.

## Streamlit Cloud

Для деплоя через Streamlit Community Cloud:

1. Создайте приватный репозиторий на GitHub.
2. Загрузите туда проект.
3. Откройте Streamlit Community Cloud.
4. Нажмите `New app`.
5. Выберите репозиторий.
6. Main file path укажите:

```text
app.py
```

7. Нажмите `Deploy`.

Если реальные выгрузки не коммитятся в GitHub, используйте загрузку файла через левую панель дашборда.
