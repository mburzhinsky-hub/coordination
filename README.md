# Контроль задач ИТО

Статический дашборд для контроля задач инженерно-технического отдела из выгрузок Битрикс24.

Работает как обычный сайт на GitHub Pages:

- без Streamlit;
- без Python;
- без сервера;
- без сборки;
- без установки зависимостей.

## Что изменено в этой версии

- Дашборд растянут на всю ширину монитора.
- Задачи со статусом `Отложена` полностью исключаются из расчетов, таблиц, рисков, загрузки людей и динамики.
- Проектные контейнеры исключаются из расчетов как самостоятельные задачи. Их подзадачи учитываются внутри соответствующих проектов.
- Ежедневные задачи Максима исключаются из расчетов:
  - `Ежедневная проверка статусов закупок/договоров`;
  - `Ежедневный контроль выполнения задач`.
- Добавлена загрузка новой выгрузки прямо через сайт кнопкой `Загрузить выгрузку`.
- После каждой загрузки строится отдельный автоотчет: KPI-дельты, управленческое резюме, журнал движения задач, топ рисков по людям и проектам.
- Добавлен раздел `Дайджесты`: персональные ежедневные списки действий по ролям — ответственный, постановщик, соисполнитель, наблюдатель.
- В дайджесты попадают только полезные пункты: просрочено, срок сегодня, ближайшие 3 дня, без срока, нет активности, ждёт контроля, высокий риск и изменения с прошлой выгрузки.
- У каждого сотрудника есть готовый текст для отправки в Telegram/Битрикс24 и кнопка `Скопировать дайджест`.
- Кнопка `Скачать CSV` в разделе дайджестов выгружает все персональные пункты по сотрудникам, ролям и задачам.
- Кнопка `Скачать отчет CSV` выгружает сводку, инсайты и журнал изменений между двумя последними срезами.
- При загрузке файла через сайт дашборд сразу открывает именно загруженную выгрузку, даже если имя файла не содержит дату.
- В проект добавлена новая выгрузка `tasks_2026-07-08_10-43-28.xls`.

## Как открыть

После загрузки репозитория на GitHub включи GitHub Pages:

`Settings` → `Pages` → `Deploy from a branch` → `main` → `/root` → `Save`.

После публикации открой ссылку GitHub Pages. Главная страница дашборда — `index.html`.

## Быстрая загрузка выгрузки через сайт

На опубликованном дашборде нажми сверху `Загрузить выгрузку` и выбери файл `.xls` из Битрикс24.

Такой файл будет обработан прямо в браузере. Это удобно для быстрой проверки без GitHub Desktop.

После загрузки сверху появится/обновится блок `Что изменилось в задачах`. Он сравнит текущий файл с предыдущей доступной выгрузкой и покажет:

- сколько задач добавилось и ушло из выгрузки;
- какие задачи стали просроченными или вышли из просрочки;
- где изменились сроки, ответственные, статусы и риск;
- какие сотрудники и проекты требуют первоочередного внимания;
- управленческое резюме короткими выводами.

Важно: загрузка через сайт сохраняет файл только в браузере пользователя. Чтобы выгрузка была доступна всем и не пропадала после очистки браузера, добавь файл в репозиторий через GitHub Desktop.

## Как добавить выгрузку в репозиторий

Новые выгрузки Битрикс24 класть в папку:

```text
data/raw/
```

Формат имени файла:

```text
tasks_YYYY-MM-DD_HH-MM.xls
```

Примеры:

```text
tasks_2026-07-03_09-00.xls
tasks_2026-07-03_14-00.xls
tasks_2026-07-04_09-00.xls
```

После добавления файла нужно открыть:

```text
data/exports.json
```

И добавить имя новой выгрузки в список `files`.

Пример:

```json
{
  "files": [
    "tasks_2026-07-03_07-04-17.xls",
    "tasks_2026-07-03_11-09-38.xls",
    "tasks_2026-07-03_14-00.xls"
  ]
}
```

## Частота обновления

Обычный режим: 1 раз в рабочий день утром до 09:30.

Активная проектная фаза: 2 раза в день — утром и после обеда.

Критический период перед сдачей: 3 раза в день — утром, в середине дня и в конце рабочего дня.

Старые выгрузки не удалять: они нужны для раздела `Динамика` и для блока `Изменения с прошлой выгрузки` внутри персональных дайджестов.

## Персональные дайджесты

Раздел `Дайджесты` строится по всем ролям из выгрузки Битрикс24:

- `Исполнитель` / `Ответственный` — что человек должен сделать сам;
- `Постановщик` — что нужно принять, проконтролировать или вернуть;
- `Соисполнители` — где нужен вклад сотрудника;
- `Наблюдатели` — только критичные риски и задачи на контроле, чтобы не спамить лишним.

Для каждого сотрудника доступен фильтр:

- `Красная зона и сегодня` — оптимальный ежедневный режим;
- `Только красная зона` — короткий список для срочного пуша;
- `Все пункты` — полный список, включая ближайшие сроки, гигиену и изменения.

Кнопка `Скопировать дайджест` копирует готовый текст сообщения. Его можно сразу отправить сотруднику в Telegram, Битрикс24 или почту.
