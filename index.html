Как обновлять дашборд

  1. В Битрикс24 открой список задач отдела.
  2. Сделай экспорт задач в файл .xls.
  3. Для быстрой проверки нажми сверху Загрузить выгрузку и выбери файл. Дашборд пересчитается прямо в браузере.
  4. Чтобы новая выгрузка была доступна после публикации на GitHub Pages для всех пользователей, положи файл в data/raw/.
  5. Открой data/exports.json и добавь имя новой выгрузки в список files.
  6. Залей изменения через GitHub Desktop: Commit to main → Push origin.
  7. Открой ссылку GitHub Pages и нажми Обновить.

Задачи со статусом Отложена, проектные контейнеры и ежедневные служебные задачи исключаются из расчетов.

Операционный Гант

Гант строится только по датам из выгрузки. Если заполнены плановые начало и окончание — показывается плановая полоса. Если есть фактический старт и дедлайн — рабочая полоса. Если есть только дедлайн — ромб-событие. Задачи без дат попадают в режим Пробелы планирования.

Используй карточки над шкалой как быстрые фильтры, группировку по проектам или ответственным, а кнопку К сегодняшнему дню — чтобы мгновенно вернуться к текущему срезу.

Персональные дайджесты

Раздел Дайджесты работает в action-only логике: задача попадает человеку только если у него есть понятное действие по роли.

Для ежедневной работы используй режим Красная зона, проверка и сегодня. Внутри каждой карточки есть только четыре смысла: сигнал, кому писать, кому мяч и что сделать.

Можно открыть мини-отчет в отдельном окне, отправить его через Печать / PDF, скачать как HTML, скопировать краткий или подробный текст, отдельную карточку задачи или скачать TXT-файл.

Наблюдателям не показываются все задачи подряд: контроль подключается с 3-го дня, красная зона приемки — с 5-го дня. Дмитрий Храпугин видит управленческие эскалации отдела.

Как часто делать выгрузки

Обычный режим: 1 раз в рабочий день утром до 09:30.

Активная проектная фаза: 2 раза в день — утром и после обеда.

Перед сдачей проекта: 3 раза в день — утром, в середине дня и в конце рабочего дня.
